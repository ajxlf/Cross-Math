What's inside

Features

Every feature was built from scratch — no game framework, no puzzle library. The generator, validator, layout engine, and timer are all hand-written.

Dynamic puzzle generation
Grids from 11×11 to 20×20 with randomised equation count and placement. No two puzzles are the same.
Crossword intersections
Horizontal and vertical equations cross at shared cells. The intersection value must satisfy both equations simultaneously.
Real-time validation
Cells turn green or red the moment an equation can be evaluated — no submit button needed.
60-second timer mode
Toggle a countdown that locks the board at zero. Timer survives device rotation via rememberSaveable.
Score tracking
One point per correctly solved equation. Score updates live and is preserved across rotation.
Rotation-safe state
ViewModel for puzzle data + rememberSaveable for UI state. Rotating the device never loses progress or resets the timer.

Puzzle generation

How the generator works

Every puzzle is generated fresh at runtime — there's no database of pre-made puzzles. The algorithm places equations like crossword entries.

1
Size the grid. If an equation count was requested, derive the grid size from it (clamped to 11–20 rows/cols). Otherwise, pick a random size in that range.
2
Place the seed equation. Retry up to 500 times to place the first equation at a valid random position and direction.
3
Grow the puzzle (85/15 split). 85% of the time, try to intersect an existing equation at one of its number cells. 15% of the time, place a fully random equation anywhere. This ratio keeps the grid dense and crossword-like.
4
Validate compatibility. isCompatible() rejects equations that are adjacent (would visually merge), share cells with mismatched values, or run parallel with no gap.
5
Blank one cell per equation. Randomly choose left operand, right operand, or result as the blank. Shared intersection cells can never be blanked.
6
Retry if needed. The whole attempt is retried up to 20 times. If all fail, falls back to a minimal random puzzle rather than crashing.
